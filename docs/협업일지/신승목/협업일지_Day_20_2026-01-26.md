## [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?
**답변**:  
```
A/B 테스트 결과 후 업종 인식률을 포함한 테스트 결과 개선을 위해 industries.yaml 파일에 키워드를 추가/제거하는 과정에서 같은 키워드가 여러 곳에 있는 것을 확인하여 직접 확인하고 중복을 제거하여 겹치지 않게 처리할 계획입니다.
사실, 세부 그룹별 주요 키워드가 의미적으로는 겹치는 부분이 꽤 있기 때문에 키워드가 겹치는 업종들은 제대로 인식되지 않을 것입니다.
다만, 일반적으로 생각하기에 납득할 수 있을 정도의 업종 인식률을 얻고, 제대로 인식되지 않은 그룹에 분류되어도 그 그룹의 키워드가 해당 세부 업종의 광고 컨텐츠 생성에 문제가 없다면 최종적으로 만족스러운 광고 컨텐츠를 얻을 수 있다고 생각합니다.
이미지 생성 프롬프트는 A/B 테스트 실행했을 때, 템플릿을 거치면 사용자 입력에 사진을 찍는 각도와 조명에 대한 설명이 추가되기 때문에 100% 승률로 더 나은 결과가 나오지만, 광고 문구의 경우 평가 시 승률이 85% 정도로 나왔습니다. 이는 업종 인식이 제대로 되지 않아서 키워드 전달을 받지 못한 것도 있고, 업종에 대한 키워드가 오히려 사용자 입력에서 나오는 핵심 단어를 가려서 점수를 낮게 받는 경우도 있었습니다.
기존 이미지 생성 프롬프트를 생성하는 config_loader.py에서 업종을 인식하지 못했을 때 S-4 항목에 배치하여 경쟁도가 높은 업종에 대응할 수 있도록 했었는데, S등급에 세분화기 이루어지면서 혼선이 발생하였습니다. 결국 S, A, B, C, D, E에 인식되지 못했을 때 F_general에 배치될 수 있도록 두 개의 industries.yaml 파일에 업종 인식에 실패하였을 때 F_general에 배정되도록 config_loader.py를 수정하였고, industries.yaml 파일에서 F_general에 대한 기본값을 넣어주었습니다.
```
