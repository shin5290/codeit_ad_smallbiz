## [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?
**답변**:
```
1. 문서화의 중요성
   - 개발 과정을 기록하지 않으면 나중에 복원 어려움
   - private_doc의 설계 결정 기록이 보고서 작성에 큰 도움
   - 문제 해결 과정도 중요한 자산

2. 메모리 최적화 전략의 핵심
   - ZIT Singleton 패턴 (항상 VRAM 상주)
   - rembg만 동적 로드/언로드 (즉시 제거)
   - 시간적 분리가 아닌 메모리 중첩 최소화

3. 3단계 락 시스템의 역할
   - _CACHE_LOCK: 컴포넌트 로드 보호
   - _PIPELINE_LOCK: 파이프라인 flush 방지
   - _EXECUTION_LOCK: GPU 순차 실행
```
