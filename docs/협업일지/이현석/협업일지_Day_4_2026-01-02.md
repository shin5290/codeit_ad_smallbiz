## [5] 오늘 분석/실험 중 얻은 인사이트나 발견한 문제점은?
**답변**:
```
주요 발견:
- LoRA vs 체크포인트 모델 차이 확인: 일부 LoRA는 특정 베이스 모델 전용
- 메모리 관리: 체크포인트 모델 3개 동시 로드는 L4 22GB에서 불가능
- 자동 언로드 시스템으로 모델 교체 방식 구현 성공
- Variant(fp16) 지원 여부가 모델마다 다름 → 자동 fallback 구현

기술적 해결:
- models/ 폴더 기반 로컬 캐싱으로 재다운로드 방지
- auto_unload=True로 생성 완료 후 즉시 메모리 해제
- variant 오류 발생 시 자동으로 재시도하는 예외 처리
```

## [6] 일정 지연이나 협업 중 어려웠던 점이 있다면?
**답변**:
```
LoRA와 체크포인트 모델 혼동으로 인한 재설계
- 초기에 LoRA로 계획했으나 실제로는 체크포인트 모델 필요
- 메모리 제약으로 인해 자동 언로드 시스템 추가 구현

HuggingFace 모델 variant 호환성 이슈
- 일부 모델은 fp16 variant 미지원
- try-except로 자동 대응 로직 추가
```
