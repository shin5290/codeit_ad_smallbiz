# 광고 생성 프로젝트 방향 제안서

## 1. 현재 문제점

1. **사용자 니즈 파악의 어려움**: 소상공인을 위한 서비스임에도 구체적인 타겟이 불명확하여 서비스 방향 설정에 한계가 있다.
2. **창의적 기능 구현의 한계**: 명확한 도메인이 없어 차별화된 기능을 기획하기 어렵다.
3. **프로젝트 진행 속도 저하**: 전반적인 개발 진행이 느리다.
4. **기능 중심의 접근**: 좋은 서비스에 대한 고민보다 기능 구현 자체에만 집중하고 있다.

---

## 2. 제안 사항

### 2.1 도메인 특정: 카페/음식점

#### 선정 근거

- **높은 광고 니즈**: 배달 플랫폼 경쟁 심화로 음식점 마케팅의 중요성이 증가하고 있다.
- **시장 성장**: 개인 카페 비율이 증가하는 가운데, 생존 경쟁이 심화되고 있으며 SNS 마케팅이 필수 요소로 자리잡았다.
- **정책적 지원**: 정부 지원사업의 주요 대상이 카페, 음식점일 정도로 지속적인 니즈가 확인된다.
- **명확한 차별화**: 니즈가 뚜렷하고 시장 규모가 크며, 다른 업종과 차별화가 용이하다.

### 2.2 플랫폼별 맞춤 광고 지원

카페/음식점 사장님들이 주로 이용하는 플랫폼(인스타그램, 네이버 플레이스, 배달의민족 등)의 광고 특성을 분석하여, 각 플랫폼에 최적화된 결과물을 제공한다.

### 2.3 이미지 생성 전략 전환: T2I → I2I 중심

#### 전환 배경: AI 생성 이미지의 신뢰도 문제

| 항목 | 수치 |
|------|------|
| AI 광고 주목도 | 51% |
| AI 광고 신뢰도 | 23% |
| AI 광고 호감도 | 16% |
| AI 광고 구매 의향 | 15% |

*출처: CJ 메조미디어(Insight M)*

국내 학술 연구에 따르면, 동일한 이미지라도 "AI 이미지입니다"라고 알렸을 때 평가가 하락하는 것으로 나타났다. 또한 한국언론진흥재단 미디어서베이(2025)에서 AI 관련 개인정보 침해 우려 비율이 88.7%에 달할 정도로, AI 콘텐츠에 대한 불안감이 높은 상황이다.

#### 제안 방향

- **T2I(텍스트→이미지)**: 보조 기능으로 활용
- **I2I(이미지→이미지)**: 메인 기능으로 전환
  - 사장님이 촬영한 실물 이미지 기반
  - 배경 적용, 카피라이팅, 구도 조정 등 편집 중심

### 2.4 챗봇 시스템 구축

마케팅의 필요성을 인지하면서도 전문 지식, 시간, 비용 등의 이유로 어려움을 겪는 소상공인을 위해 챗봇 시스템을 제안한다.

**주요 기능**:
- 업종별 트렌드 분석
- 맞춤형 마케팅 전략 제안
- 실시간 Q&A 지원

**기술 스택**: 컨텍스트 엔지니어링, RAG(Retrieval-Augmented Generation) 활용

### 2.5 확장성 고려

카페/음식점 도메인을 먼저 완성도 있게 구축한 후, 단계적으로 다른 업종으로 확장한다.

## 3. 플랫폼별 광고 특성 분석

### 3.1 종합 비교

| 구분 | 인스타그램 | 네이버 플레이스 | 배달의민족 |
|------|------------|-----------------|------------|
| 주 목적 | 브랜딩, 감성 소통 | 정보 제공, 신뢰 구축 | 즉시 전환, 주문 유도 |
| 타겟 연령 | 20-30대 (Z세대 70%) | 전 연령 (92% 사용) | 20-40대 |
| 콘텐츠 소비 | 하루 1회 이상 (48.1%) | 방문 전 검색 (78%) | 식사 시간대 집중 |
| 주요 지표 | 저장 > 공유 > 좋아요 | 클릭률, 전화/길찾기 | 썸네일 클릭 → 주문 전환 |

### 3.2 이미지 특성

| 플랫폼 | 이미지 성격 | 촬영 스타일 | 색감/톤 |
|--------|-------------|-------------|---------|
| 인스타그램 | 감성적, 여백 활용, 스토리텔링 | 자연광, 플랫레이, 라이프스타일 연출 | 파스텔 톤, 밝고 따뜻함, 필터 활용 |
| 네이버 플레이스 | 정보 전달 중심, 깔끔하고 명확 | 정면 촬영, 전체 구도, 밝은 조명 | 자연스러운 색감, 과도한 보정 지양 |
| 배달의민족 | 식욕 자극, 클로즈업, 비주얼 임팩트 | 음식 중심, 김 연출 | 생생한 색상, 대비 강조, 윤기 강조 |

### 3.3 광고 문구 특성

| 플랫폼 | 문구 길이 | 톤앤매너 | 핵심 요소 | 예시 |
|--------|-----------|----------|-----------|------|
| 인스타그램 | 짧음 (5-15자) | 감성적, 스토리텔링, 이모지 활용 | 해시태그 15개+, 위치 태그, 궁금증 유발 | "오늘 같은 날엔 ☕ 따뜻한 라떼 한 잔 #성수카페 #힐링" |
| 네이버 플레이스 | 중간 (10-20자) | 정보 전달, 신뢰 구축, 키워드 최적화 | 검색 키워드, 위치 정보, 영업시간/메뉴 | "강남역 3번 출구 브런치 카페 주차 가능" |
| 배달의민족 | 매우 짧음 (3-10자) | 직접적, 행동 유도, 혜택 명시 | 할인 금액, 배달 시간, 리뷰 별점 | "치킨 1+1 10분 배달 ★4.8" |

### 3.4 광고 시스템 비교

| 구분 | 인스타그램 | 네이버 플레이스 | 배달의민족 |
|------|------------|-----------------|------------|
| 광고 형식 | 피드, 스토리, 릴스, 캐러셀 | 플레이스 광고, 지역소상공인 광고, 배너 | 오픈리스트, 울트라콜, 배민1, 중문배너 |
| 과금 방식 | CPC/CPM, 예산 자유 설정 | CPC, 지역소상공인 0.5원/노출 | CPC, 일 단위, 위치별 차등 |
| 타겟팅 | 관심사, 행동 패턴, 위치 | 지역, 검색 키워드, 카테고리 | 앱 내 행동, 주문 이력, 지역 |
| 콘텐츠 수명 | 스토리 24시간, 피드 반영구적 | 상시 노출 (검색 기반) | 식사 시간대 집중, 실시간 변동 |
| 핵심 목표 | 팔로워 증가, 브랜드 인지도 | 전화 문의, 방문 유도 | 즉시 주문, 매출 전환 |

## 4. 추가 기능 제안

| 기능 | 설명 |
|------|------|
| **영수증 기반 마케팅 리포트** | 종이 영수증 사진 업로드 → OCR로 시간대별/메뉴별 판매 분석 → AI 기반 광고 전략 및 문구 제안 |
| **짧은 영상 생성/편집** | 숏폼 콘텐츠 트렌드에 맞춘 영상 광고 제작 지원 |
| **메뉴판 → 광고 변환** | 메뉴판 이미지 업로드 → OCR로 메뉴 추출 → 각 메뉴별 광고 카피 자동 생성 |
| **리뷰 기반 광고 문구 생성** | 기존 고객 리뷰에서 긍정 키워드 추출 → 신뢰도 높은 광고 문구 생성 |

---

## 5. 결론

카페/음식점이라는 명확한 도메인을 설정하고, 플랫폼별 특성에 맞춘 광고 생성 서비스를 구축할 것을 제안한다. AI 이미지 생성의 신뢰도 한계를 인식하여 I2I 방식을 메인으로 전환하고, 챗봇 시스템을 통해 마케팅 지식이 부족한 소상공인도 쉽게 활용할 수 있는 서비스를 목표로 한다.
