# 발표자료 폴더
- 발표 자료 준비를 위한 이미지
- 발표 스크립트 작성 내용(.md)
- 발표 자료
- 프로그램 시연 자료

# 보고서 폴더
- 각자 진행 상황에 대한 결과를 협업 일지 외에 보고서로 남길 때 활용할 수 있는 폴더
    - 파일 이름 끝에 작성자 명을 붙여서 각자 관리
- 최종 보고서: 각자 진행했던 부분을 마크다운 형식으로 작성
    - 파일명: 발표 순서에 맞게 앞에 숫자를 붙이고 파일 이름 끝에 작성자 이름을 붙여서 보고서 폴더에 올린다.
    - 팀장이 병합(ex. 1. 프로젝트개요_발표_신승목.md)

# 협업일지 폴더
- readme에 링크 첨부
- 작성 템플릿 제공, 매일 작성한 협업일지를 올려둘 것
- (예정) 역할 분배 후 개인 프로젝트 체크리스트.md를 추가하여 진행 상황 반영

# 회의록 폴더
- 아침 첫 시간에 디스코드로 전체 회의를 진행
- 진행한 회의 내용을 회의록으로 저장하여 전체 진행상황 공유

# doc 폴더
- 전체 시스템에 대한 안내 문서를 마크다운으로 저장
- (예정) 프로젝트 체크 리스트.md : 전체 내용 작성 후 각 구성원의 체크리스트로 분할
- (예정) 시스템 개발 설계서.md
- (예정) 시스템 아키텍처 설계서.md
- (예정) 시스템 디렉토리 구조.md
