# Git 규칙
1. src 폴더 내의 `.py` 파일은 **항상** 버그가 없는 것을 확인한 뒤, `git push`를 수행한다.
2. `git push` 전에는 **항상** `git pull`을 먼저 수행하여 파일의 충돌이 일어나지 않게 한다.
3. `git push`를 수행할 때에는 **항상** `git add .` 대신 `git add dir/dir/file.ext` 형태로 파일을 지정해서 push한다.
4. 로컬 **alpha** 브랜치에서 작업을 수행하고, **origin/alpha** 브랜치를 연결하여 `git push` 한다.

# 유지보수하기 좋은 코드
1. 읽기 쉽고, 일정한 규칙과 흐름(메인 로직)을 유지해야 한다.
2. 함수는 한 가지 기능만 수행해야 한다.
3. 같은 톤의 함수들을 묶어서 배치한다.
4. 디버그 로깅, 예외 처리는 메인 스토리와 구분하여 한 곳에서 다루게 한다.

# 함수명 짓기
1. 함수명은 가능하면 동사로 시작하고, 함수 호출 시 어떤 의미가 있는지를 알려주어야 한다.
2. (함수 이름) + (내부 코드) + (기능)이 일관되어야 한다.
3. 함수는 반복 코드를 재활용하기 위해 사용하는 것이기 때문에 사용자가 무슨 함수인지 명확히 알고 호출할 수 있어야 한다. (변수에 대해 Type Hinting과 설명이 필요하다.)
4. AND, OR 들어가면 나쁜 함수 이름일 가능성이 높다. 사용자 입장에서 의미를 전달해 주어야 하고, 해당 함수는 분할할 필요성이 있을 수 있다.
